顺序容器概述

容器 数据结构 访问 特征
array 固定大小数组 快速随机 不能添加或删除元素
vector 可变大小数组 快速随机 在尾部增删快，中间增删慢可能很慢
string 字符串数组 快速随机 类似 vector
deque 双端队列 快速随机 头尾插入删除快
list 双向链表 双向顺序 插入删除块
forward_list 单向链表 单向顺序 链表任何位置插入速度快

vector、string：

  1. 连续的内存空间
  2. 中间位置增删元素耗时
  3. 有时候，添加一个元素，可能要额外分配存储空间，这时需要把每个元素都移动到新空间里。

list/fowward_list：

  1. 任意位置增删快
  2. 不支持随机访问
  3. 内存开销比 vector/deque/array 大

deque：

  1. 类似 vector/string，支持快速随机访问，中间位置增删代价高

新标准的容器比旧版本快很多。

现代 C++程序应该使用标准库容器，而不是原始的数据结构。新标准容器的性能几乎与精心优化过的同类数据结构一样好。

赋值和 swap

swap 交换两个相同类型的容器。swap 不对任何元素进行拷贝、删除和插入，只是交换容器的内部数据结构。时间在 O(n)。这意味着，指向容器的迭代器、引用、指针在 swap 之后均不会失效。

swap 两个 array 会真正交换。

vector 对象是如何增长的

shrink_to_fit 只适用于 vector/string/deque，退出不需要的多余空间。将 capacity 调整为 size 相同大小。只是请求，并不保证一定退还内存。
capacity/reserve 只适用于 vector/string
reserve(n) 分配 n 个元素的内存空间，并不改变容器中元素的数量，仅仅影响预先分配的内存空间。永远不会减少容器的占用内存。调用 reserve 之后，capacity 会大于等于 n。

策略：减少容器控件重新分配次数的策略。当需要扩容的时候，vector、string 分配一个更大的内存空间。

vector 每次重新分配内存空间时，要移动所有元素。采用策略后，扩长操作通常比 list、deque 还快。

size 和 capacity：capacity 是容量，size 是元素数目。具体分配多少额外空间要看标准库具体实现。

vector 实现策略：每次需要分配新内存空间时，将当前容量翻倍。

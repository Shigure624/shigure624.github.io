定义和改变shared_ptr的其他方法

shared_ptr<T> p(q, d); // p接管了内置指针q指向对象的所有权。使用可调用对象d，代替delete函数
shared_ptr<T> p(p2, d); // 同上，只不过p2是智能指针的拷贝
p.reset() // p是唯一指向其对象的智能指针的话，reset释放此对象
p.reset(q) // 如果传递q，令p指向q，否则p置为空
p.reset(q, d); // 如果额外还传递d，用来delete

不要混合使用普通指针和智能指针

拷贝shared_ptr会增加引用计数。

void process(shared_ptr<int> ptr) {
  // 使用ptr，此时引用计数值至少为2
} // ptr离开作用域，销毁。但是引用计数值至少不是0
// 正确用法
shared_ptr<int> p(new int(42));
process(p); // 拷贝p，引用计数增加
int i = *p;
//错误用法：混用普通指针和只能指针
int *x(new int(1024));
process(x); // 错误，不能用普通指针隐式转换shared_ptr
process(shared_ptr<int>(x)); // 正确，但是内存会被释放，之后再用x就有问题了。
int j = *x; // 这样就有问题，x是悬空指针

也不要使用get初始化另一个智能指针，或给智能指针赋值

get返回一个内置指针，指向管理的对象。get的用法是在不能使用智能指针的代码传递一个内置指针。
get返回的指针不能delete

shared_ptr<int> p(new int(42)); // 引用计数：1
int *q = p.get(); // right，但是用q的时候注意别让他释放
{
  shared_ptr<int>(q);
} // 程序块结束，q被销毁。p成了选空指针
int foo = *p; // 错误，未定义：p指向的内存已经被销毁了。

补充：shared_ptr

A std::shared_ptr owns the object it points to but, unlike std::unique_ptr, it allows for multiple references. A special internal counter is decreased each time a std::shared_ptr pointing to the same resource goes out of scope. This technique is called reference counting. When the very last one is destroyed the counter goes to zero and the data will be deallocated.
This type of smart pointer is useful when you want to share your dynamically-allocated data around, the same way you would do with raw pointers or references.

unique_ptr

一个unique_ptr拥有它所指向的对象。
初始化必须采用直接初始化形式，没有类似 make_shared 的函数。
不支持拷贝或赋值操作。

unique_ptr<T> u1;
unique_ptr<T, D> u2 // D是个释放指针的可调用对象

unique_ptr<T, D> u(d); // 空的unique_ptr，指向类型为T的对象
u = nullptr
u.release() // 放弃控制权，返回指针，u置为空
u.reset() // 释放u指向的对象
u.reset(q) // 用q指向这个对象，q是内置指针
u.reset(nullptr)

std::unique_ptr<int> p;
std::unique_ptr<int> p(new int(42));

std::unique_ptr<Type> p = std::make_unique<Type>(...size or parameters...);
std::unique_ptr<int> p2 = std::make_unique<int[]>(1024);

用完以后自动析构、删除内存。不允许复制，最多有一个 unique_ptr 指向对象。这是为了防止被多次删除。

所有权转移：

unique_ptr<string> p2(p1.release()); // release 将p1置为空，所有权从p1转到p2
unique_ptr<string> p3(new stirng("Tres"));
p2.reset(p3.release()); // reset释放p2原来的内存，所有权从p3到p2

release会切断unique_ptr和管理对象的联系。通常在初始化另一个智能指针的时候，使用release返回的指针。

weak_ptr

一种不控制所指向对象的智能指针。指向由一个shared_ptr管理的对象。把 weak_ptr 绑定到 shared_ptr 上，不增加 shared_ptr 的引用计数。

std::shared_ptr<int> p_shared = std::make_shared<int>(100);
std::weak_ptr<int> p_weak1(p_shared);
std::weak_ptr<int> p_weak2(p_weak1);

weak_ptr<T> w
weak_ptr<T> w(sp)
w = p // p可以是shared、weak
w.reset // 置空
w.use_count() // w共享对象shared_ptr的数量
w.expired() // use_count为0
w.lock() // 若expired true，返回空的shared_ptr，否则返回指向w的shared_ptr

A std::weak_ptr is basically a std::shared_ptr that doesn't increase the reference count. It is defined as a smart pointer that holds a non-owning reference, or a weak reference, to an object that is managed by another std::shared_ptr.

This smart pointer is useful to solve some annoying problems that you can't fix with raw pointers. We will see how shortly.

指针操作

补充：std::shared_ptr 作为参数传递的时候，到底应不应该用引用?

void do_job(std::shared_ptr<my_object > obj);还是 void do_job(std::shared_ptr<my_object > &obj);

第二种，别的线程把管理的对象销毁了会出问题吧?

都可以，前者表示你想增加一个 owner；
后者表示你想重用之前的 owner，并且自己保证在使用 obj 的时候前一个 owner 是有效的
如果你既不想增加一个 owner，又没法保证在使用的时候前一个 owner 还活着，用 weak_ptr
传 const shared_ptr<T >&的话，那就不如直接传个裸指针。
别的使用 unique_ptr<T >的场景也可以方便的 call 这个函数了。
简单地说，传引用的话，会线程不安全，因为传入的 shared_ptr 可能会在其他线程 reset, 那样的话， do_job 函数访问 obj 时会有问题. 比如下面的 sample, 使用 TestSharePtrRef 时程序会挂掉.
传 value 的话，引用计数会+1, 线程安全。当然如果是在单线程环境或者 owner/lifetime 明确的情况下，可以用传引用，毕竟共享指针传值的开销不小(引用计数和保证线程安全的开销)
实际上很多情况下如果用引用的话，用的是 const std::shared_ptr<my_object >& obj
我觉得从语义来说，一般情况下应该是不引用更好吧，减少麻烦事情，这个应用场景应该就是它的设计初衷之一吧
因为它主要就是一个指针，本身占用的资源不多，传递起来应该比较轻快

补充：max_element（）及min_element（）

二者返回的都是迭代器或指针。

头文件：#include<algorithm>

求数组的最大值或最小值

  1. vector容器，例 vector vec

最大值：int maxValue = *max_element(v.begin(),v.end());

最小值：int minValue = *min_element(v.begin(),v.end());

  1. 普通数组，例 a[]={1,2,3,4,5,6};

最大值：int maxValue = *max_element(a,a+6);

最小值：int minValue = *min_element(a,a+6);

求数组最大值最小值对应的下标

  1. vector容器，例 vector<int> vec

最大值下标：int maxPosition = max_element(v.begin(),v.end()) - v.begin();

最小值下标：int minPosition = min_element(v.begin(),v.end()) - v.begin();

  1. 普通数组，例a[]={1,2,3,4,5,6};

最大值下标：int maxPosition = max_element(a,a+6) - a;

最小值下标：int minPosition = min_element(a,a+6) - a;

注意：返回的是第一个最大（小）元素的位置。

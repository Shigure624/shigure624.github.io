左值和右值：
在C中比较简单：左值可以放在赋值语句左侧，右值不能。
C++中：
1. 左值表达式的求值结果是一个对象或者一个函数
2. 常量表达式不能用在左值
3. 当一个对象被用作右值的时候，用的是对象的值；对象用作左值的时候，用的是对象的地址

隐式转换

隐式转换是系统跟据程序的需要而自动转换的：

  1. 比int小的类型，整形提升
  2. 条件中：非布尔值转为布尔值
  3. 初始化：初始值转成变量的类型，赋值语句中，右侧运算对象转换成左侧运算对象
  4. 算数运算、关系运算的运算对象转成同一种
  5. 函数调用

算术表达式隐式转换顺序为：

  1. char - int - long - double

  2. float - double

  3. 算术表达式

    int m = 10;
double n = m; // n = 10.0;隐式把m转为double类型

    int m = 10;
float f = 10.0;
double d = m + f;// n = 20.0; 隐式把m和f转为double类型

    short thirty;
long solong = thirty; // 扩展后的值在solong中，为long类型，thirty值不变。
    转换 潜在问题
    double 转 float 精度降低，值可能超出范围
    浮点转整型 小数部分丢失，值超出范围
    long 转 short 通常只复制右边的字节 
    0->bool // 0 is false
非0v->bool // v is true
int t = 3.88; // t=3;

算数转换

提升：promotion vs conversion: promotion 是指同类型的转换，比如 bool/char/short 之类的，转为 int，或者 float 转 double

整形提升 integral promotion

First, let’s examine the automatic conversions.When it evaluates expressions, C++ converts bool, char, unsigned char, signed char, and short values to int. In particu- lar, true is promoted to 1 and false to 0.These conversions are termed integral promotions.
To execute the statement on line 3, a C++ program takes the values of chickens and ducks and converts both to int.Then the program converts the result back to type short because the answer is assigned to a type short variable.You might find this a bit round- about, but it does make sense.The int type is generally chosen to be the computer’s most natural type, which means the computer probably does calculations fastest for that type.

  1. 将 bool/char/unsigned char/signed char/short 转为 int。true->1 false->0。
  2. 如果 short 比 int 短，unsigned short 将被转换为 unsigned int。
  3. wchar_t 被提升成以下中第一个宽度足够存储 wchar_t 取值范围的类型：int/unsigned int/long/unsigned long。
  4. int 和 float 相加，转成较大的类型。比如 9.0/5=1.8，5 被转换为 double 类型。
short chickens = 20;
short ducks = 35;
short fowl = chickens+ducks; // 程序将chickens和ducks转成int，再赋给short

通常将 int 类型选择为计算机最自然的类型，这意味着计算机使用这种类型时，运算速度可能最快。

ANSI C 与 C++相同
K&R C：float + float 将 float 提升为 double

A prvalue of an integer type other than bool, char16­t, char32­t, or wchar_­t whose integer conversion rank ([conv.rank]) is less than the rank of int, can be converted to a prvalue of type int, if int can represent all the values of the source type; otherwise, the source prvalue can be converted to a prvalue of type unsigned int.
A prvalue of an integer type can be converted to a prvalue of another integer type. A prvalue of an unscoped enumeration type, can be converted to a prvalue of an integer type.

显式转换

显式转换也叫强制转换，自己主动让这个类型转换成别的类型。

C-style cast 是 (type)value
Function-style cast type(value)

cast-name:
static_cast : 有关联的 对象之间能够进行转化.
const_cast : 忽略 const/volatile 修饰, 但是实际不会修改对象, 常用于重载
dynamic_cast : 运行时类型识别，最多用于向下转化, 但并不安全
reinterpret_cast : 二进制间, 可无关联对象间的转化, 非常不安全.

static_cast

int m = 5;
char c = (char)m;//显式把 m 转为 char 类型

double d = 2.0;
int i = 1;
i += static_cast<int>(d);//显式把 d 转换为 int 类型

具有明确定义的类型转换，不包含底层const都可以使用。

const_cast

只能改变运算对象的底层const。不能改变类型

const char* pc;
char *p = const_cast<char*>(pc); // 正确，但是不要通过p写值。

reinterpret_cast

位模式提供较低层次上的重新解释。

int *ip;
char *pc = reinterpret_cast<char*>(ip);

*pc指向int，而不是字符。

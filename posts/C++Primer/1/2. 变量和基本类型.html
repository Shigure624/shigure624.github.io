The standard does not specify if plain char is signed or unsigned.
In fact, the standard defines three distinct types: char, signed char, and unsigned char. If you #include <limits.h> and then look at CHAR_MIN, you can find out if plain char is signed or unsigned (if CHAR_MIN is less than 0 or equal to 0), but even then, the three types are distinct as far as the standard is concerned.
Do note that char is special in this way. If you declare a variable as int it is 100% equivalent to declaring it as signed int. This is always true for all compilers and architectures.

std::cout << CHAR_MIN; 本机是-128

char 用作数组下标时，一定要先转 unsigned char （因为 char 通常是有符号的）。不能直接转 int 或 unsigned int ，会数组下标越界。

wchar

In C++, wchar_t is a distinct fundamental type (and thus it is not defined in <cwchar> nor any other header).

In C, this is a typedef of an integral type. 定义：typedef short int wchar_t;

wchar vs wchar_t

wchar_t is a distinct type, defined by the C++ standard. WCHAR is nonstandard, and as far as I know, exists only on Windows. However, it is simply a typedef (or possibly a macro) for wchar_t, so it makes no practical difference.

浮点型

float 和 double 的表示范围和精度：float 占用 4 个字节，也就是 32 个比特。double 占用 8 个字节，也就是 64 个比特。float 和 double 的范围是由指数的位数来决定的。

位：bit

类型 符号位 指数位 尾数位
float 1 8 23
double 1 11 52
范围

float 的指数范围为-127--128，double 的范围是-1023-1024。

负指数决定了绝对值最小的非零数，正指数决定了绝对值最大的数。也即决定了范围。

也即 float 的范围为-2128-2128,double 的范围是-21024-21024。

精度

float 和 double 的精度是由尾数位决定的。浮点数在内存中是按照科学计数法来存储的，其整数部分始终是一个隐藏着的 1。由于他是不变的，因此对精度不会造成影响的。

float 精度范围是：2^23 一共 7 位，因此最多能表示 7 位，但是能保证的是 6 位。

double 的精度范围是 2^52 一共 16 位，同理最多能表示 16 位，但是能保证的是 15 位。

double 的比较

浮点数可以用 >=和<=进行比较。浮点数（double）之所以不能用 == 和 != 是因为浮点数在内存中都是按精度来存储的，所以你看到的 3.12 也许本来是 3.123456 取的小数点后 2 位位精度，所以当与另一个 3.12 进行比较时可能会出现很多种情况，如果另一个 3.12 其实是 3.121111 取的小数点后 2 位精度的话，那么两个数你看着相等，其实是不等的。
所以对浮点数进行比较的时候，一般都是相减然后去和一个精度比较。
例如：
a=3.12; b=3.13;
当 a 和 b 比较时就应该：

#define N 0.0001
if(a-b<N)

就说明 a 小于 b;.
而不是直接去比较 if( a<b)

对于 double 类型，比如 double d1=0.0000001，double d2=0d 当判断两个数据 d1 和 d2 是否相等的时候，一般不直接使用
if(d1==d2)
由于计算机在表示小数时都有误差，我们一般认为偏差在一个较小的范围内则两个数据是相等的。
正确的代码如下：

public static void main(String[] args) {
double dis=1e-6;
double d1=0.0000001d;
double d2=0d;
System.out.println(d1==d2); //直接判断为 flase
System.out.println(Math.abs(d1-d2)<dis); //允许一定的误差范围，判断结果为 true
}

引用

C++与 C 的一个区别。

  1. 引用是一个对变量的别名，不是一个真的对象，是个小名。引用和对象本身是绑定关系，所以改变了引用，也就改变了对象。
  2. 引用必须在声明时初始化，因为引用需要和一个值绑定在一起。初始化的必须是对象，不能使字面值，不能使用表达式计算结果。int &r = 10; // 错误
  3. 引用和被绑定的变量，他们的类型必须严格相同。（但是有两个例外，在 const 那块说）
  4. 引用只能绑定一次，不能更改绑定关系。

指针

  1. 本身就是一个对象
  2. 允许对指针赋值、拷贝
  3. 生命周期内，可以先后指向几个不同的指针，引用不行，引用可以 r=j 这样，但是改变 j 不会改变 i，这样只是赋值。
  4. 指针在定义时不用赋初始值，引用必须。

指针和指向的对象类型需要严格匹配，但是有两种情况例外。p56 p534

补充：指针的引用

That is taking the parameter by reference. So in the first case you are taking a pointer parameter by reference so whatever modification you do to the value of the pointer is reflected outside the function. Second is the simlilar to first one with the only difference being that it is a double pointer. See this example:

void pass_by_value(int* p)
{
    //Allocate memory for int and store the address in p
    p = new int;
}

void pass_by_reference(int*& p)
{
    p = new int;
}

int main()
{
    int* p1 = NULL;
    int* p2 = NULL;

    pass_by_value(p1); //p1 will still be NULL after this call
    pass_by_reference(p2); //p2 's value is changed to point to the newly allocate memory

    return 0;
}

理解复合类型的声明

一条定义语句可能定义出不同类型的变量。

int i = 1023, *p = &i, &r = i;

* 和 &没有传播性。所以定义一个不会影响后面的变量。

引用本身不是个对象，所以不能定义指向引用的指针。

const 限定符

  1. const 限定符，修饰的变量，不能改变，必须在定义时初始化。
    （注意！在 C 中，可以不显示地初始化，默认是：局部变量随机，外部 0。试验了一下貌似都是 0。）

  2. 用一个对象初始化另一个对象，这两个是不是 const 都无所谓。这是一种拷贝。

int i = 42;
const int ci = i;
int j = ci;

编译过程中替换。const 在同一文件中有效。不同文件中定义相同 const 变量属于不同值。如果希望多个文件用，在声明中都加 extern，这样可以只定义一次，多次声明。

const 的引用

对常量的引用，不能用来修改它所绑定的对象，常量也不可以通过引用修改值。

const int ci = 1024;
const int &r = ci; // right
r = 42; // wrong，不能用r去修改ci
int &r2 = ci // wrong，不允许给ci赋值，所以也不能通过引用修改ci。假设可以的话，就可以通过改r2修改ci，这样不对。

术语：常量引用，指对 const 的引用。也就是上面 r。

引用的类型需要严格一致，但是有两个例外：

  1. 初始化常量引用时，允许用任意表达式作为初始值，前提是这个表达式的结果能转换成引用的类型。所以是针对常量引用 const &，一般引用不可以这么干。
int i = 40;
const int &r1 = i; // right
const int &r2 = 42; // right
const int &r3 = r1 * 2; // right
int &r21 = 42; // wrong，前面说过了
int &r31 = r1 * 2; // wrong，前面说过了

const int i = 42;
const int &r = i; // ok
r = 10;// illegal
int &r1 =i;// illegal

引用和所引用对象类型必须一致的例外情况：初始化常量引用时，允许用任意表达式作为初始值。

double d = 3.14;
int &r1 = d; // wrong

./test_2.cpp:41:10: error: non-const lvalue reference to type 'int' cannot bind to a value of unrelated type 'double'
    int &r1 = d;

如果是 const 就可以

double d = 3.14;
const int &r = d;

编译器中间产生一个临时量，用来存储 d 生成一个临时的 int

const int temp = dval;
const int &ri = temp;

常量引用，其引用的对象本身是否是常量，并没有限定。

指针和 const

一个技巧：从右向左读

char * const cp; ( * 读成 pointer to ) // cp is a const pointer to char
const char * p;  // p is a pointer to const char;
char const * p; // 同上

pointer to const const double *p = &pi; 指针 p 不能用于改变 pi 的值，但是 pi 可以通过 pi 变量修改。

const pointer int *const pi = &px; 指针的值不可改变。

指针的类型和所指对象必须类型一致，有种例外：指向常量的指针，指向非常量对象，可以不一致

初始化：

  1. const * 可以不初始化
  2. *const 必须初始化

顶层 const 和底层 const

顶层：表示指针本身是 const，也就是*const，此外，算数类型也是顶层，没有 const 修饰的引用应该是顶层
底层：表示指针指向的对象是常量，也就是const*，用于声明引用的 const 都是底层 const。底层：非 const 可以转为常量，反之不行。

补充

当执行对象的拷贝过程中（赋值操作，函数的值传递）时，如果被拷贝对象拥有底层 const 资格，则拷贝对象必须拥有相同的底层 const 资格。
如果拷贝对象拥有底层 const，则无所谓被拷贝对象是否有 const。
总结：对于一个既是顶层 const 又是底层 const 对象来说，无所谓它是否为顶层 const，只要关注它的底层 const 就行了。

const int p = 5;
//无外部链接
extern const int temp;// 有外部链接的const 声明，在其他文件使用
int temp = 2;
const int pi = 3.14;
const int& a = temp; //合法，但temp改变会引起a改变
const int& b = 2; //合法，编译器自动创建一个临时变量，然后将临时变量赋值给b
int& c = pi; //不合法，c的值可以改变，但pi是const，与程序员本意矛盾
int& d = 2; //不合法，因为不是const int&，编译器并不会创造一个临时变量来赋值

const 变量 = 非 const 变量 可以
非 const 变量 = const 变量 可以
const 引用 = 非 const 引用 可以
const 引用 = const 引用 可以
非 const 引用 = 非 const 引用 可以
非 const 引用 = const 引用 不可以！
非 const 指针 = const 指针 不可以！同理

constexpr

常量表达式 const expression：是指不会改变，并且在编译时就能得到计算结果的表达式。

常量表达式 eg：

  1. 字面值 literal 是常量表达式。
  2. const int sz = get_size(); 这个不是常量表达式，因为这个是函数 get_size 运行时才知道的结果。

编译时得到计算结果的常量表达式。C++11 新标准引入，由编译器验证结果是否是常量表达式。如果用常量表达式函数，函数也需要通过 constexpr 定义。

字面值：一些比较简单、容易得到的值。算数类型、引用、指针都属于字面值类型。类不属于字面值类型
constexpr 指针：初始化只能用 nullptr 或者 0，或者指向函数外的变量，因为函数内的变量地址会变化。另外，这个 constexpr 指针，仅仅对指针有效，对所指的对象无关。

constexpr int *p = nullptr; // const pointer，pointer is const
const int *q = nullptr; // point to const int.

补充 typedef vs define

  1. typedef is limited to giving symbolic names to types only, whereas #define can be used to define an alias for values as well, e.g., you can define 1 as ONE, 3.14 as PI, etc.
  2. typedef interpretation is performed by the compiler where #define statements are performed by preprocessor.
  3. #define should not be terminated with a semicolon, but typedef should be terminated with semicolon.
  4. #define will just copy-paste the definition values at the point of use, while typedef is the actual definition of a new type.
  5. typedef follows the scope rule which means if a new type is defined in a scope (inside a function), then the new type name will only be visible till the scope is there. In case of #define, when preprocessor encounters #define, it replaces all the occurrences, after that (No scope rule is followed).

auto

自动推测类型。如果auto i = 0, *p = &i; i 是整型，p 是指向整型的指针。

  1. auto 用在引用上：就是引用变量所对应的那个对象的类型

    int x = 0, &cr = x;
auto i = cr; // i是int

  2. auto 会忽略顶层 const，如果想指定 const，需要加上 const

    int i = 0;
const int ci = i, &cr = ci;
auto b = ci; // b 是整数
auto c = cr; // c 是整数
auto d = &i; // d 是指针，指向整型
auto e = &ci; // e 是 指针，指向整型常数

    const auto f = ci; 指定 f 是 const 类型。

  3. 可以将引用类型设置为 auto

    const int ci=i;
auto &g = ci;
auto &h = 42; // 错误
const auto &j = 42; // 正确

decltype 类型指示符

用于推测变量类型，推测的方式是：通过表达式，且并不计算表达式

decltype(f()) sum = x; sum 的类型是 f 的返回值。

decltype 保留顶层 const。

decltype 保留引用类型。

decltype 对括号的处理

decltype((i)) 有双层括号之后，是 int&类型。
